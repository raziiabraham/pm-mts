## Five-minute walkthrough

1. State the user signal, evidence, and product question.
2. Define the smallest useful output and explicit exclusions.
3. Name relevant product and repository surfaces, constraints, ownership, and specialist review.
4. Write product behavior and verification separately from implementation instructions.
5. Record open questions and create one read-only discovery prompt before a planning or implementation prompt.

## Take it to your product

Create a one-page context brief that another contributor can carry across tools without reconstructing the product reason from chat history.

## Ask an AI coding agent

> Read this guide and the Portable Context Brief. Interview me section by section. Keep the user signal, evidence, objective, behavior, scope, constraints, output, verification, and unknowns distinct. Then draft one read-only discovery prompt and one bounded planning prompt. Do not implement anything until I approve the brief.

## Evidence boundary

Clear context improves delegation but does not verify the result. Generated plans, prose, code, or diagrams remain hypotheses until checked against product behavior, source evidence, technical tests, and human expertise.
