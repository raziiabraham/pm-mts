# Chapter 15: When a Feature Becomes an Agent

## Product question

> What goal, tools, memory, budgets, stops, evaluation, and human authority make this agent bounded enough to operate?

## Five-minute walkthrough

1. Decide whether the product is a pipeline, tool-using feature, or agentic loop.
2. Define the goal and the product behavior the loop may pursue.
3. List allowed tools, inputs, outputs, memory, retrieval, and skills.
4. Set time, cost, step, permission, escalation, and stop conditions.
5. Define trajectory checks, outcome evaluation, exposure, rollback, and how greater agency must be earned.

## Take it to your product

Write one Agent Charter and identify its riskiest line: the action, tool, memory, or authority that would create the greatest user or operational consequence if wrong.

## Ask an AI coding agent

> Read this guide and the Agent Charter. Help me distinguish a deterministic pipeline, tool-using feature, and agent. Interview me about goal, tools, memory, skills, budgets, stops, permissions, trajectory evaluation, and human escalation. Challenge every action that changes external state and require an earned-agency path.

## Evidence boundary

An agent charter makes the intended harness inspectable; it does not prove the implementation enforces those limits or that the model behaves reliably. Technical, security, privacy, safety, and operational verification remain required.
