## Five-minute walkthrough

1. State the user job and the role the image plays in the product.
2. Define input type, size, dimensions, metadata, permissions, and validation.
3. Map upload, processing, storage, variants, CDN delivery, caching, and deletion.
4. Name pending, retry, failure, replacement, and recovery states visible to the user.
5. Write a bounded handoff that asks an AI coding agent or engineer for evidence, tests, and trade-offs—not merely implementation.

## Take it to your product

Rewrite one vague image requirement as a problem-first asset contract. Review it with engineering, design, security, and operations where those responsibilities apply.

## Ask an AI coding agent

> Read this guide and the Image Requirement Record. Help me turn one image feature into an inspectable contract. Ask about the user job before architecture. Separate user-visible requirements from implementation options, identify missing lifecycle states, and mark every security, privacy, cost, or infrastructure question that requires specialist review.

## Evidence boundary

The figures and worksheet teach a reasoning structure. They do not prescribe one universal media architecture or prove that a proposed storage, CDN, compression, or processing design fits a real system.
